# Contributing to SIZEM / poprua-cras

Thanks for contributing.

## Setup

Follow the Docker flow in the [README](README.md). Use `.env.example` — never commit secrets.

## Pull requests

- Target `main` unless the issue says otherwise
- Include tests for behavior changes when practical
- Run `php artisan test` and `vendor/bin/pint --dirty` before opening the PR
- Keep commits focused; describe *why* in the PR body
